FUNCOES DE CRIAÇÃO DE XML SAT DA BIBLIOTECA E1_SAT.

Funções reponsaveis pela geração dos xmls do sat para operações de venda e cancelamento.
Abaixo é apresentado um fluxograma a ser seguido pela autoamação no processo de geração do xml para o SAT.

AbreCupomVenda Esssa função deve ser a primeira a ser chamada pela aplicação para iniciar a criação do xml de venda para o SAT.

Parâmetros
path_fileCaminho absoluto onde o arquivo xml será gerado.
EX:

C:/sat/venda_0001.xml

versaoDadosEntVersão do leiautedo arquivo dedados do AC. (Ex: 0.07 ou 0.08).

TAG: A03
Parametro obrigatório
Tipo: Numérico
Tamanho: 1-4
Casas decimais: 2

CNPJ_SHCNPJ Software House.
Deve ser informado apenas números. Caracteres (ponto, barra, hífen, etc.) serão removidos.
Caso o CNPJ tenha menos que 14 digitos a tag será preenchida com 0 não significativos.
Nos casos de teste fim-a-fim de ativação, completar com 14 dígitos “0”(zero).

TAG: B11
Parametro obrigatório
Tipo: Numérico
Tamanho: 14

signACAssinatura do Aplicativo Comercial
Assinatura de (CNPJ Software House + CNPJ Emitente) que gerou o CF-e.
Nos casos de teste fim-a-fim de ativação, completar com 344 dígitos “0”(zero).

TAG: B12
Parametro obrigatório
Tipo: Alfanumérico
Tamanho: 1-344

numeroCaixaNúmero de 0 a 999 referente ao Caixa em que o SAT está conectado.

TAG: B14
Parametro obrigatório
Tipo: Numérico
Tamanho: 3

CNPJ_emitCNPJ do emitente.
Deve ser informado apenas números. Caracteres (ponto, barra, hífen, etc.) serão removidos.
Caso o CNPJ tenha menos que 14 digitos a tag será preenchida com 0 não significativos.

TAG: C02
Parametro obrigatório
Tipo: Numérico
Tamanho: 14

IEDeve ser informado apenas caracteres alfanumérico. Caracteres (ponto, barra, hífen, etc.) serão removidos.
Caso IE tenha menos que 12 digitos a tag será preenchida com espaços a direita.

TAG: C12
Parametro obrigatório
Tipo: Numérico
Tamanho: 2-14

IMInscrição Municipal.
Este campo deve ser informado, quando ocorrer a emissão de CF-e conjugada, com prestação de serviços sujeitos ao ISSQN e fornecimento de peças sujeitos ao ICMS.

TAG: C13
Essa TAG não é obrigatória e para que não seja incluida no XML deve-se passar parametro vazio.

'S' - Desconto sobre subtotal será rateado entre os itens sujeitos ao ISSQN.
'N' - Desconto sobre subtotal não será rateado entre os itens sujeitos ao ISSQN.

TAG: C16

AbreCupomVenda2 Essa função cria o arquivo de vanda organizando os arquivos a partir do diretório informado em 'path_root'.
.

A Estrutura criada deve ser: /XML/SAT/VENDA/yyyy/MM/dd/yyyyMMddhhmmss.xml
onde:

yyyy = ANO
MM = MÊS
dd = DIA
hh = HORA
mm = MINUTO
ss = SEGUNDO

Parâmetros
path_rootCaminho raiz onde a estrutura de pasta e o arquivo será gerado.
Observação
Os demais parametros são identicos aos da função AbreCupomVenda
Retorna
O retorno dessa função deve ser o caminho absoluto onde o arquivo foi gerado.
