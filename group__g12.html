Descrição detalhada

E1 Bridge.

Este módulo tem como objetivo fornecer comunicação entre ambiente desktop (Windows/Linux/Android) e SmartPos Elgin Pay,
sendo possível realizar operações de venda utilizando uma biblioteca que faz comunicação com o terminal de pagamento via TCP/IP.

Componentes

E1_Bridge (DLL/SO/AAR) - Extensão do aplicativo (DLL/SO/AAR) onde estão exportadas todas as funções para Iniciar uma venda ou operação administrativa no terminal de pagamento SmartPOS.
E1_DigitalHub (antigo E1_bridge) (APK) - Aplicativo android a ser instalado no SmartPOS para configurações de porta e ativação do serviço para comunicação com Windows.
Este componente está disponível na loja Elgin. Você pode habilitá-lo na loja Elgin, na seção Token.
e1_bridge_configs (JSON) - Arquivo de configuração da DLL gerado no workdir caso usuário nao especifique a variável de ambiente e1_bridge_configfile.

Por onde começar

  1. O primeiro passo é habilitar o app para que ele possa ser carregado no seu terminal. Para isso, acesse seu perfil de software house no link abaixo:
    https://elgin.app.br/
  2. Após acessar, no menu lateral você deve acessar a opção Token e então habilitar o aplicativo conforme imagem abaixo:
  3. No terminal, registrado com o token configurado anteriormente, você poderá realizar o download do aplicativo acessando o menu Elgin Store e clicando sobre o APP,
    vide imagem abaixo:
  4. Com o app instalado você poderá começar a realizar as transações.
    • É importante que você verifique as configurações do app antes de começar os testes.
    • Para isso abra o aplicativo e role a página para cima. Um botão deve aparecer no rodapé, pressione-o e digite a senha de acesso às configurações (987654321)

Arquivo de Configuração

e1_bridge_configfile - Nome da variável de ambiente para setar o path do arquivo de configuracao.
Caso o usuário não especifique a variável de ambiente, o arquivo e1_bridge_configs.json será criado no workdir.
Abaixo o exemplo de um arquivo de configuração gerado na pasta workdir.

{
"ipTerminal": "192.168.0.1",
"portaStatus": 3001,
"portaTransacao": 3000,
"senha": "",
"senhaHabilitada": false,
"timeoutResposta": -1
}
